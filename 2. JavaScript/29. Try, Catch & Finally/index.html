Try, Catch & Finally

Introducción

En esta ocasión realizaremos el espacio de corrección de errores llamado try, catch & finally para javascript. Este entorno de trabajo es muy útil para que nuestro programa continue a pesar de que se halla generado un error. A continuación lo veremos más en detalle para saber como es su correcto funcionamiento.

Sintaxis

try { x = 10; }

Primero comenzaremos con el bloque de try, donde, el codigo busca y captura el posible error que pueda contener. Si esto sucede, lo guarda dentro de una variable (no la debemos declarar) que la pasamos como parámetro dentro de la siguiente sintaxis que corresponde a catch. Si pretamos atención, la declaración que hemos hecho nos retornaría un error, por lo cual, esto entraría en la sección de catch que vamos a construír a continuación.

try { x = 10; } catch (error) { console.log(error); } console.log('El programa continua a pesar del error, gracias al fragmento de try y catch.');

En el fragmento de código anterior, hemos declarado una variable que nos retorna el error, y esta, va a ser procesada por el fragmento de catch donde podemos ver el resultado en la consola:

Podemos observar 2 situaciones en el mensaje impreso en consola. Primero, nos define el tipo de error que contiene la variable y segundo nos retorna el mensaje del error, es decir: ReferenceError (tipo de error) y x is not defined (mensaje del error).

Traduciendo el mensaje nos está indicando que es un error de referencia y en el mensaje nos dice que la variable 'x' no está definida. Eso no es todo lo que debemos observar, si prestamos atención notaremos que a pesar del error, el programa continúa su flujo imprimiendo el mensaje correspondiente en la línea del mensaje.

Existe un bloque más que podemos agregar a esta sentencia y es conocido como finally. Este bloque de código al declararlo siempre se va a ejecutar, independientemente del bloque de código que se encuentre en try.

try { x = 10; } catch (error) { console.log(error); } finally { console.log('Termina la revisión de errores.'); } console.log('El programa continua a pesar del error, gracias al fragmento de try y catch.');
