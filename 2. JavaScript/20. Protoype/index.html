Prototype

Los Prototype son funciones orientadas a objetos. La declaración es la misma que utilizamos cuando creamos una función general, lo que cambia en su sintaxis es que a sus parámetros los llamamos como en los objetos con this, así que al momento de llenar los parámetros tome esos datos que estamos agregando al momento de la llamada.

Debemos saber que aún no hemos declarado la function prototype, solo dimos una pequeña introducción de como armar una función para que pueda usarse luego con prototype.

Sintaxis:

function Tiburon(nombre, nombreCientifico) {
this.nombre = nombre;
this.nombreCientifico = nombreCientifico;
}

Una vez construida la functión debemos agregar los methods con prototype. Osea estamos creando una method function para la clase Tiburon. Este method se llamará "atacar".

Tiburon.prototype.atacar = function() {
console.log("cuidado fuera del agua")
}

Como podemos obserbar primero, se coloca el nombre de la clase function que es "Tiburon" luego, colocamos el method prototype y por último colocamos el nombre que tendra este method dentro de la clase que en este caso es "atacar".
A esta declaración que hemos armado nos queda declararle "=" que es una function y dentro de las llaves de function colocaremos el proceso de este method.

Instanciando la function class Tiburon

Ahora para utilizar los unos atributos específicos de la nueva clase que hemos hecho debemos instanciarla. Para ello crearemos una nueva variable con nombre "martillo" y le instanciamos "Tiburon."

const martillo = new Tiburon('tiburon martillo', 'prionacle glauca');

Herencia en los Prototypes

Podemos heredar los Prototypes de una clase, y la sintaxis es muy similar a lo que venismos construyendo con las herencias de los objetos. Debemos saber que siempre tenemos que crear una clase antes de hacer una herencia.

Sintaxis:

Para esta sintaxis crearemos un ejemplo creando una clase nueva llamada "TiburonToro" (recordemos que los nombres de las clases siempre están en mayúsculas). Esta clase tendrá atributos heredados (nombre, nombreCientifico) y atributos nuevos (peligrosidad).

Para comenzar esta nueva construcción vamos a declarar que es una function (recordemos que los prototypes trabajan en base a este tipo de construcción de objetos) con nombre "TiburonToro" y dentro de los parametros los atributos que heredamos y nuevos que son: nombre, nombreCientifico y peligrosidad. Recordemos que primero se colocan los atributo que vamos a heredar y luego los nuevos atributos. Esta parte de la sintaxis quedaría de la siguiente manera:

function TiburonToro(nombre, nombreCientifico, peligrosidad) {}

A continuación con la palabra clave this porque estamos haciendo referencia a este constructor, y super, debemos colocar de qué clase vamos a heredar los prototypes, en nuestro caso es de la clase "Tiburon". Pero, un detalle muy importante es que en esta instancia se declara como un valor como si fueramos a copiar el valor de una variable:

this.super = Tiburon

Luego de realizar el paso anterior sí colocamos los atributos que vamos a heredar de la clase tiburon que en nuestro caso son "nombre" y "nombreCientifico".
Debemos colocar nuevamente this y super pero en este caso, si pasar como parámetro los atributos que vamos a heredar:

this.super(nombre, nombreCientifico)

Por último debemos crear el atributo interno de peligrosidad en la clase para que tome el valor del parámetro. La construcción de esta es como la venimos trabajando anteriormente:

this.peligrosidad = peligrosidad

El código hasta este punto quedaría de la siguiente manera:

function TiburonToro(nombre, nombreCientifico, peligrosidad) {
this.super = Tiburon
this.super(nombre, nombreCientifico)
this.peligrosidad = peligrosidad
}

Instancia de TiburonToro

Luego de construir la herencia debemos proceder a crear la instancia de Tiburon Toro la cual le debemos dar el prototype como en los casos anteriores.

La instancia se llamará "costero" y debemos tener en cuenta que al darle la instancia debemos colocar en su sintaxis prototype, ya que necesitamos heredar específicamente esa parte de la clase tiburon:

Debemos prestar de todas maneras mucha atención en la sintaxis, ya que, debemos pasar 2 valores en la misma linea de código. Porque, primero a "costero", le vamos a pasar el prototype de TiburonToro.prototype que a su vez le vamos a pasar el valor que es una nueva instancia de "tiburon". Osea, que tenemos que copiar los datos de Tiburon.

const costero = TiburonToro.prototype = new Tiburon()

Ahora nos queda darle acceso a la clase tiburon para que termine de heredar los datos:

TiburonToro.prototype.constructor = Tiburon

Sobreescritura de Métodos

Esta práctica es muy útil cuando necesitamos heredar un method pero debemos hacer pequeños cambios en su contenido.

Para nuestro ejemplo, sobreescribiremos el method que le hemos dado a la clase "Tiburon" para darle un contenido más específico a "TiburonToro".

Sintaxis:

La construcción de la sobreescritura es muy sencilla, simplemente debemos hacer la llamada de la clase que estamos utilizando (en nuestro caso de ejemplo es "TiburonToro") y procedemos a acceder al prototype luego, al nombre de la function (en nuestro ejemplo el nombre de function es: atacar) y por último sobreescribir el contenido de function. Esto se hace de la siguiente manera:

TiburonToro.prototype.atacar = function() {
console.log('Cuidado fuera del agua, toro en las proximidades') }

Si prestamos atención simplemente lo que acabamos de hacer es literalmente llamar al prototype y sobreescribirlo.
