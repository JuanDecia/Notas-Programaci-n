Javascript

Introducción e historia

Introducción

JavaScript se usa como un lenguaje de programación para el navegador Web, cuando se corre un código escrito en JS el navegador a través del interprete del lenguaje, ejecuta ese código por lo que se le considera una tecnología de Front-End. JavaScript se usa para agregar dinamismo a las páginas Web. Este lenguaje principalmente fue formulado para validar formularios, es decir, dado un formulario debemos dar instrucciones de JavaScript en las cuales vamos a poder validar los campos con reglas que este necesite. También, tiene el propósito de poder interactuar con las páginas a través del uso de botones que al presionarlo generen una acción, tal como el despliegue de una caja de diálogo o la ejecución de una operación llamado a otro procedimiento o función.


El Frontend es la principal función del lenguaje, y como JS no es compatible con todos los navegadores se ha desarrollado estándares que indican el conjunto de indicaciones a ejecutar, las cuales deben ser implementadas por todos los navegadores o mejor dicho por todos los interpretadores del lenguaje.


Historia

La historia de JavaScript comienza en 1995, cuando Brendan Eich, un ingeniero de software de Netscape, fue contratado para crear un lenguaje de programación que pudiera ser utilizado en el navegador web Netscape Navigator.


Inicialmente, el lenguaje fue llamado Mocha, pero luego fue renombrado a LiveScript y finalmente a JavaScript, para capitalizar la popularidad del lenguaje de programación Java en ese momento.


JavaScript se lanzó junto con Netscape Navigator 2.0 en septiembre de 1995 y rápidamente se convirtió en uno de los lenguajes de programación más populares en el mundo de la tecnología.


En 1997, la empresa Netscape se unió con la organización de estándares web, la W3C, para crear un estándar para JavaScript, conocido como ECMAScript. ECMAScript fue diseñado para garantizar que el lenguaje fuera interoperable entre diferentes navegadores y plataformas.


En 1998, se lanzó la especificación de ECMAScript 2.0, que incluía nuevas características como la gestión de excepciones y la creación de objetos personalizados. Desde entonces, se han lanzado varias versiones de ECMAScript, cada una con nuevas características y mejoras.


En 2009, Ryan Dahl creó Node.js, un entorno de tiempo de ejecución de JavaScript que permite que el lenguaje se ejecute en el servidor, lo que permitió la creación de aplicaciones web altamente escalables y de alto rendimiento.


En los años siguientes, JavaScript continuó evolucionando y creciendo en popularidad. En 2015, se lanzó ECMAScript 6, que introdujo características importantes como las clases, los módulos y las promesas, y ayudó a establecer a JavaScript como un lenguaje de programación maduro y completo

Funcionamiento de JavaScript

JS un lenguaje de alto nivel, interpretado y dinámico, es decir, no necesita un compilador para poder ejecutarse y sus instrucciones se pueden generar en tiempo de ejecución real. Para JS existe un programa que se llama interprete que lee línea a línea el código (de izquierda a derecha) y lo va traduciendo a medida que lo va leyendo en el navegador del usuario. Cuando se carga una página web que incluye código JavaScript, el navegador descarga y ejecuta el código para realizar las tareas programadas por el desarrollador.


JavaScript funciona interactuando con el Document Object Model (DOM), que es una interfaz que representa la estructura de la página web en el navegador. El DOM se compone de nodos que representan los elementos de la página, como encabezados, párrafos y botones, y los desarrolladores pueden utilizar JavaScript para acceder y manipular estos nodos.


Además, JavaScript también puede interactuar con el objeto global del navegador, que incluye cosas como el historial del navegador, la pantalla, el tiempo de espera y más. Los desarrolladores pueden utilizar JavaScript para manipular estos objetos globales y realizar tareas como redireccionar a otra página, mostrar mensajes emergentes y crear efectos visuales.


También es capaz de comunicarse con el servidor web, lo que permite a los desarrolladores crear aplicaciones web dinámicas y en tiempo real. Los desarrolladores pueden utilizar JavaScript para enviar y recibir datos a través de tecnologías como AJAX (Asynchronous JavaScript and XML) y WebSockets.

JavaScript: Lenguaje debilmente tipado

En programación, los lenguajes de programación se dividen en dos categorías principales: lenguajes tipados y lenguajes no tipados. Los lenguajes tipados son aquellos en los que cada variable tiene un tipo de datos específico y debe ser declarada con ese tipo de datos. Por otro lado, los lenguajes no tipados, también conocidos como lenguajes débilmente tipados, son aquellos en los que las variables no están asociadas con un tipo de datos específico y pueden cambiar de tipo durante la ejecución del programa.


JavaScript es un lenguaje débilmente tipado por lo cual no es necesario declarar los tipos de las variables explícitamente, esto significa que las variables no tienen un tipo de datos fijo y pueden cambiar de tipo durante la ejecución del programa. Por ejemplo, una variable que inicialmente contiene un número puede cambiar su valor a una cadena de texto en cualquier momento del programa.


Este enfoque flexible de tipos de datos tiene ventajas y desventajas. Por un lado, hace que sea fácil escribir código más rápido, ya que no es necesario declarar explícitamente los tipos de datos de las variables. Por otro lado, también puede aumentar el riesgo de errores en tiempo de ejecución y dificultar la detección de errores durante la fase de desarrollo.


Es un lenguaje multiparadigma, tiene instrucciones para trabajar como un lenguaje funcional. También, tiene instrucciones para funcionar como un lenguaje orientado a objetos mezclando varios modelos de programación incluyendo la programación imperativa, por eso se analizarán diferentes formas de programar.


También, puede utilizarse en aplicaciones híbridas que se pueden programar con varias tecnologías. La importancia de JS recae en el hecho que utiliza ampliamente en una basta cantidad de páginas Web, sobre todo aquellas que son dinámicas convirtiéndose en el lenguaje que proporciona dinamismo a la web.

Integrar JS a nuestro sitio web

Existen dos maneras de integrar JS en HTML. Estas maneras son muy similares, ya que, en ambas utilizaremos la etiqueta <script>. Una de las maneras es a traves del archivo de tipo HTML, es decir, declaramos las etiquetas y dentro del elemento de las etiquetas comenzamos a escribir código JS. La otra manera es utilizando un archivo externo de tipo '.js' como lo hacemos con los archivos de CSS, esta última es la manera ideal de crear archivos JS.

Dentro del archivo HTML

<script>
// Código Javascript dentro del archivo HTML.
</script>

Vincular archivo externo

La segunda forma, es vincular un archivo externo de javascript utilizando el atributo 'src' de nuestra etiqueta <script>:


<body>
// Contenido HTML
<script src='./scripts/main.js'> </script>
</body>

Debemos tener en cuenta el siguiente aspecto de esta estructura, la etiqueta <script> se debe encontrar al final de la construcción del archivo HTML justo antes de donde se cierra la etiqueta de body. Esto es debido a que como el lenguaje es interpretado va leyendo de arriba hacia abajo, entonces gracias a esta lógica, primero carga la página y luego carga las funciones que hemos dado con Javascript.


Si la etiqueta <script> se encuentra en la parte superior de nuestro código (al comenzar la etiqueta body o dentro de head), la página puede generar errores. Esto es debido a que al cargarse primero las funciones de js pueden no detectar nodos que deben ser utilizados por las mismas, por esta misma razón, es que puede generar algún error de funcionalidad en la página.

Consola

Introducción

Los navegadores tienen una consola integrada en el navegador en donde podemos ver los resultados de nuestro código hechos en JS. Para visualizar esta consola, debemos abrir el navegador y haciendo click derecho y luego en la opción inspeccionar, o también, una manera más rápida de abrirla es con la tecla F12. abrimos el explorador para desarrolladores. También, existen extensiones para VS Code en donde podemos ver estos resultados dentro del mismo sin tener que pasar al navegador, esta extensión es llamada Quoka.


Para ejecutar un fragmento de código y visualizarlo, debemos colocar la palabra clave “console.log()” que dentro del parámetro (paréntesis) colocaremos lo que se mostrará en consola.

Ambiente y Depuración

El ambiente de desarrollo de JS nos permite manejar y detectar errores al mismo tiempo que nos facilita la tarea de creación o modificación de contenido de nuestro código.


Usualmente se desea ver el estado de la aplicación en determinado momento, para este caso es que se usa la función de JavaScript console.log(), que nos imprime los resultados pasados en sus parámetros en la consola del navegador. Al visualizar la consola, podemos ver el mensaje o resultado que hemos pasado en los parámetros de console.log ().
