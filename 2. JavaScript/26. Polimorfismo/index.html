POO: Polimorfismo

Introducción

El concepto de polimorfismo se aplica especialmente cuando estamos trabajando con POO y las clases. Este concepto es para resumir líneas de código en la cual la declaración puede trabajar de manera diferente según para la clase en la que se está aplicando. Es decir, si tenemos dos clases con métodos diferentes podemos crear una declaración que al llamarla trabaje de una forma si la llamamos para la clase 1 y que trabaje de otra forma si la llamamos para la clase 2. Esto se debe porque ambas clases tienen métodos diferentes.

Tomando en cuenta las clases del ejemplo anterior:

Clase empleado

class Empleado { constructor (nombre, sueldo) { this._nombre = nombre; this._sueldo = sueldo; } obtenerDetalles() { return `Empleado nombre: ${this._nombre}, sueldo: ${this.sueldo}.`; } }

Clase Genrente

class Gerente extends Empleado { constructor(nombre, sueldo, departamento) { super(nombre, sueldo); this._departamento = departamento; } obtenerDetalles() { return `Gerente: ${super.obtenerDetalles()}, Depto: ${this._departamento}.`; }

Aplicando el Polimorfismo

Teniendo en cuenta las clases anteriores, ahora vamos a crear una funcion fuera del scope de las clases, en la cual, al llamarla vamos a poder aplicar el method de cada una, según, la clase que seleccionemos como parámetro. La función sería la siguiente:

function imprimirDatosClase(clase) { console.log( clase.obtenerDetalles() ); };

Con ésta función declarada podremos llamar a los métodos de la clase padre y clase hija, según necesitemos. Para verificar si el resultado es el correcto crearemos la siguiente declaración.

imprimirDatosClase(gerente);
