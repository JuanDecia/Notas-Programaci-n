Programación Orientada a Objetos

Introducción

Este lenguaje está orientado a prototipos que es un tipo específico de programación orientada a objetos. En esta ocasión usaremos la sintaxis de ES6.

La POO es una abstracción de los objetos de la realidad plasmado en nuestro código y es aquella programación donde el sistema que estamos implementando se representa como un conjunto de objetos interactuando entre sí. Para ello, necesitamos identificar los mismos y que hacen dichos objetos. Esto se logra a través del encapsulamiento que refiere a que: cada objeto es independiente. En el sentido en que cada uno tiene definido sus funciones y propiedades, de allí la noción de cápsulas. Cada una de las cuales contiene los elementos del objeto que determinan su comportamiento, este contenido de las cápsulas, está oculto al mundo exterior solo permitiendo que se vea lo que sea requerido. Por ejemplo: una persona (objeto) lleva a reparar su teléfono móvil (objeto).

Conceptos Claves

Clases

Las clases se representan como el tipo general del objeto, englobando los elementos comunes (comportamiento) que tiene un objeto. Por ejemplo: La clase “persona” que es un tipo general de lo que puede ser un objeto persona (Juan). La persona tendría definido lo que tienen las personas en general: cabeza, cuerpo, manos, pies…

Objeto

Por otra parte tenemos el objeto que es la instancia de la clase, por ejemplo: Un objeto de la clase persona puede ser “Luis”, que es una instancia de “persona (clase)” (persona (clase) => Luis (Instancia de persona, es decir, objeto)). Entonces “persona” es la clase y “Luis” es el objeto.

Atributo

Cada objeto tiene atributos que son las propiedades de dicho objeto. Si “Luis” es un objeto, las propiedades serían: altura, nombre, apellido.

Métodos

Los métodos son las funciones que determinan un comportamiento de ese objeto (caminar (), hablar (), etc.). Anteriormente hemos visto cómo agregar atributos, en este caso, un método puede ser un atributo y para ejemplificar este concepto vamos a crear un método con nombre “presentarse()” para que el objeto persona pueda colocar su “nombre y apellido”.

let persona = { nombre: ‘Juan’, apellido: ‘Decia’, edad: 28, nombreCompleto: function(){ return this.nombre + ‘ ‘ + this.apellido; };

Como podemos observar usamos una palabra clave llamada “this”, esta palabra clave es muy útil para asociar a los valores del propio objeto. Esto sucede porque “this” es un puntero hacia los atributos que están dentro del objeto, nos retornará un error si apuntamos con “this” hacia una variable que esté relacionada con un ámbito global.

También, observemos que la función tiene un return para que nos devuelva la concatenación de los valores de nombre y apellido cuando hagamos la llamada de este método. Para realizar este paso con éxito podemos colocar esta línea de código:

console.log(persona.nombreCompleto()); // Resultado: Juan Decia

Abstracción de un objeto

Un aspecto importante a resaltar en el paradigma es la abstracción, que consiste en definir qué elementos de la realidad vamos a implementar, de forma tal, que podamos ejecutar funciones requeridas para cumplir el objetivo del sistema que estamos desarrollando. Por ejemplo, si definimos el objeto “Luis” que tiene “último nivel de educación” como atributo, un dato como: “sistema de control de masa corporal” no es importante, por lo cual, no lo debemos definir como un atributo que necesitamos implementar. Si no, buscar uno acorde a la implementación del objetivo.

En términos de abstracción debemos decidir las clases que vamos a tener, los objetos que debemos definir, qué métodos debemos implementar y que atributos debemos incluir para cada objeto.

let toby = {color: "blanco", ladrar: function()}

Tenemos una variable “tobby” que es un objeto específico que “representa un perro”. Es de color “blanco” y tiene la función de ladrar ().

Método constructor

Hay un método especial conocido como método constructor que se define en cada clase, son altamente recomendables usarlos y se llaman generalmente cuando se inicializa el objeto. Se utiliza para inicializar atributos que son generales entre los objetos. Debemos saber que constructor tiene un parámetro el cual le podemos agregar elementos los cuales podemos utilizar como variables locales con el método this.

constructor() { this.color = "blanco"; }
let toby = new Perro(); // tobby.color --> retorna "blanco"

Herencia

Otro concepto importante es la herencia esto significa que una clase puede heredar otra en particular sus atributos y métodos lo que establece una jerarquía de clases. Podemos decir que la clase “perro” se extiende de la clase “animal”. Esto quiere decir que hay una cosa que tienen todos los animales en común. Esto se conoce como “generalización”. Tal es el caso del “color”, es por ello que podemos separar los atributos comunes de los atributos particulares de cada clase.

Polimorfismo

También tenemos el concepto de polimorfismo cuando hay una jerarquía de clase con herencias. Un método puede correr de distintas maneras, es decir, su ejecución es distinta de acuerdo a la subclase donde se encuentre, por ejemplo:

Conclusión

En resumen, en la POO es importante definir las clases, sus métodos, la interacción que significa como una clase llama a los métodos o atributos de otra. Esto se logra enlazando atributos o cuando un método instancia a otra clase.
