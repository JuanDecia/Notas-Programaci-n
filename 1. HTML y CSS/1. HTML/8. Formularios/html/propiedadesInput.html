Dentro de las etiquetas input podemos agregarle propiedades que modificaran su comportamiento dentro del campo, estas son adicionales al tipo de dato a la que se les está solicitando: name, id, disable, readonly, max length, max, min, value, step, placehorlder.

Name o Id:

Las propiedades name e ID trabajan de manera similar. La propiedad name sirve para nombrar una etiqueta para poder reconocer mejor el tipo de dato que se esta tomando, en el caso de que estemos pidiendo un dato de texto para el nombre o apellido de una persona, esta propiedad recibe el valor de “nombre”.
La propiedad id lo que hace es darle un identificador al input para que pueda ser modificado o conectado a una propiedad de CSS o JavaScript, es recomendable darle un valor que este relacionado al tipo de dato que se está pidiendo. En el caso de los inputs también se utiliza para que el atributo for de los label apunte hacia ese mismo nombre y así puedan asociarse.

Disable:

Este atributo sirve para deshabilitar un campo y así impide que se ingresen datos. Este atributo puede tomar dos valores true or false.

Readonly

Este atributo es algo parecido a disable, la diferencia es que no se puede habilitar, siempre permanece deshabilitado y sirve para mostrar cuadro de entrada que son solo de lectura y no se pueden editar.

Maxlength

Le especifica al navegador cuál es el máximo números de caracteres que puede dejar ingresar en el cuadro de entrada.

Max

Este atributo se usa con el tipo number, indica cual es el valor máximo que se puede ingresar en el cuadro de entrada y no un número de caracteres.

Min

Indica cual es el valor mínimo que puede ser ingresado y no un numero de caracteres

Value

Es asignado para darle un valor por defecto al elemento input. Es utilizado en caso de que la persona olvide llenar este cuadro de entrada y necesitemos dejar un valor específico que debemos guardar. En otras palabras retorna el valor que contiene el input. Debemos tener mucho cuidado con autocompletar este campo ya que es el dato que se guardará en la base de datos.

Si omites el atributo value, los datos del formulario enviado utilizarán el valor por defecto, que es on, lo cual no resulta útil. Por lo que el atributo value debe establecerse a un valor que identifique la opción claramente.

Step

Cuando tenemos un cuadro de entrada numérico tenemos dos opciones para llenarlo.
La primera es completando el campo ingresando el valor manualmente.
La segunda disponemos de flechas de incremento o decremento que podemos utilizarlas hasta llegar al valor deseado. Si no declaramos el incremento de step este se hará de a 1 unidad

Placeholder

Se usa para dar una pista al usuario sobre que debe escoger en el input. El mensaje aparecerá dentro del cuadro de texto, mientras esté vacio

Un elemento input puede tener todos los atributos que sean necesarios, estos funcionan en compañía. Debemos tener en cuenta también que no todos los atributos son soportados en los navegadores, si deseamos saber eso podemos ingresar a la web caniuse.com
