Introducción

La etiqueta <input /> es una importante herramienta que nos permite capturar distintos tipos de datos, podemos llamarlo también como tipos de campos de entrada. Un campo de entrada es un elemento en un formulario que permite al usuario ingresar información como texto, números, fechas, archivos, entre otros. La etiqueta <input /> solo recibe propiedades que van a darle la información al navegador de que tipo de entrada de dato es la que debe colocar el usuario en el campo del input.

<input type='text' name='nombre' id='nombre' />

Atributo type

Los tipos de datos que el usuario va a ingresar en el campo de la etiqueta input deben ser especificados en un atributo llamado type que es propio de la etiqueta y solo puede recibir un valor, es decir, solo podemos requerir un tipo de entrada a la vez. A continuación, comenzaremos a explorar los diferentes tipos de valores que podemos colocar en esta propiedad para poder visualizar el comportamiento del campo en el navegador.

Type: Diferentes campos de entrada

Type: Text

La propiedad 'text' en HTML es utilizada para crear un campo de entrada de texto en un formulario. Es el tipo más común de campo de entrada, y permite al usuario ingresar cualquier tipo de texto, como nombres, direcciones, comentarios, etc. La estructura básica de un campo de texto es la siguiente:

<label for='nombre'>Ingrese su Nombre: </label>
<input type='text' name='nombre' id='nombre' placeholder='Ej: Juan' />

Como nota adicional (única vez) notemos que agregamos una etiqueta llamada label, recordemos que esta etiqueta es útil para vincular el texto con el campo. Para conocer más acerca del comportamiento de dicha etiqueta les recomiendo ingresar al siguiente link: Ir a label

Type: Number

La propiedad type='number' en HTML se utiliza para crear un campo de entrada de números en un formulario. Este tipo de campo se utiliza para recopilar información numérica, como edad, teléfono, cantidades, etc.

<label for='edad'>Ingrese su edad: </label>
<input type='number' name='edad' id='edad' placeholder='Ej: 29' />

En el ejemplo podemos ver que la estructura es similar al de texto, pero además los inputs de tipo number pueden tener atributos como min y max para determinar el rango válido de números, entre otros.

Propiedad Min de Type Number

Indica cuál es el valor mínimo que puede ser ingresado. El tipo ingresado de este campo debe ser int (5) por lo que sí requerimos o auto completamos con un tipo de dato string ('5') nos arrojaría un error.

<label for='numeroMin'>Ingrese un valor menor a 50: </label>
<input type='number' min='50' id='numeroMin' placeholder='A partir del número 50' />

Propiedad Max de Type Number

Esta propiedad se usa con el tipo number, indica cual es el valor máximo que se puede ingresar en el cuadro de entrada El tipo ingresado de este campo debe ser int (5) por lo que sí requerimos o auto completamos con un tipo de dato string ('5') nos arrojaría un error.

<label for='numeroMax'>Ingrese un valor hasta 50: </label>
<input type='number' max='50' id='numeroMax' placeholder='Hasta el número 50' />

Propiedad Step de Type Number

Cuando tenemos un cuadro de entrada numérico tenemos dos opciones para ingresar los datos. La primera es ingresando manualmente en el campo el valor deseado. La segunda, disponemos de flechas de incremento o decremento que podemos utilizarlas hasta llegar al valor deseado. La propiedad step en este campo nos permite seleccionar cada cuantos números se realiza este incremento en el caso de que deseamos utilizar las flechas auxiliares que se encuentran a la derecha del campo numérico.

<label for='decadas'>Ingrese cuantas decadas avanzar: </label>
<input type='number' id='decadas' />

Type: Email

El valor email en la propiedad type, se utiliza para crear un campo de entrada de correo electrónico en un formulario. Este tipo de campo se utiliza para recopilar direcciones de correo electrónico del usuario.

<label for='email'>Ingrese un email: </label>
<input type='number' min='50' id='numeroMin' placeholder='A partir del número 50' />

Type: Password

El valor password en la propiedad de type en HTML se utiliza para crear un campo de entrada de contraseña en un formulario. Este tipo de campo se utiliza para recopilar contraseñas o información confidencial del usuario, como contraseñas de inicio de sesión o información financiera. Lo que hará este campo es que no se muestren los caracteres cuando el usuario escribe.

<label for='password'>Ingrese contraseña: </label>
<input type='password' id='password' placeholder='Ingrese Contraseña' />

Type Color

Esta propiedad permite al usuario escojer y definir un tipo de color con valores hexadecimales.

<label for='color'>Elija un Color: </label>
<input type='color' id='color' />

Lo que podemos realizar con este efecto es capurar el valor que el usuario escoge y dinámicamente cambiar el color de un contenedor o algún elemento. Para este caso vamos a cambiar el color de éste párrafo.

Type Date

Crean un campo de entrada que le permite al usuario introducir una fecha, que puede ser tanto como una caja de texto para validar el campo como una interfaz especial que le permite escoger una fecha.

<label for='fecha'>Elija una fecha: </label>
<input type='date' id='fecha' />

Type Datetime

Representa un campo para una fecha y hora global que recolecta datos con información de zona horaria. Este tipo de campos es particularmente útil para la recolección de puntos en el tiempo universales, que son equivalentes en cualquier parte del mundo. Por ejemplo, la fecha y hora de una reunión online debería ser recolectada por este control, de modo que sea exactamente la misma para todos los participantes.

<label for='dateTime'>Elija una fecha: </label>
<input type='datetime' id='dateTime' />

Type Datetime-local

Crea un campo para que el usuario pueda escoger una fecha y hora. Los valores de este campo incluyen el año, el mes, el dia y la hora.

<label for='dateTimeLocal'>Elija una fecha: </label>
<input type='datetime-local' id='dateTimeLocal' />

Type File

Permite al usuario cargar uno o más archivos desde su dispositivo a la página.

<label for='cargarArchivo'>Cargue un archivo: </label>
<input type='file' id='cargarArchivo' />

Type Search

Este campo es funcional cuando los usuarios quieren identificar textos en un sitio determinado.

<label for='buscar'>Buscar: </label>
<input type='search' id='buscar' />

Type Reset

Este tipo de input se comporta como un botón que al presionarlo, podemos limpiar todos los campos que hemos llenado en el formulario donde este se encuentra linkeado.

<label for='limpiar'>Limpiar formulario: </label>
<input type='reset' id='limpiar' />

Type Button

Representa un botón sin una acción predefinida asociada, lo que significa que por defecto, el botón no producirá ningún efecto cuando sea presionado. Estos botones son normalmente utilizados con programas del lado cliente que son ejecutados cuando el usuario los presiona.

<input type='button' value='Boton Enviar Formulario' />

Type Submit

Representa un botón sin una acción predefinida asociada, lo que significa que por defecto, el botón no producirá ningún efecto cuando sea presionado. Cuando a este botón se le define una función es para enviar los datos que captura el formulario a una base de datos.

<input type='submit' value='Submit' />

Propiedades de la etiqueta Input

Introducción

Dentro de las etiquetas input podemos agregarle propiedades que modificaran su comportamiento dentro del campo, estas son adicionales al tipo de dato a la que se les está solicitando: name, id, disable, readonly, max length, max, min, value, step, placehorlder.


Un elemento input puede tener todos los atributos que sean necesarios. Debemos tener en cuenta también que no todos los atributos son soportados en los navegadores, si deseamos saber eso podemos ingresar a la web caniuse.com.

Propiedad Name o Id

Las propiedades name o ID trabajan de manera similar. La propiedad name sirve para nombrar una etiqueta para reconocer el tipo de dato que se esta tomando. Por ejemplo, en el caso de que estemos pidiendo un dato de texto para el nombre o apellido de una persona, esta propiedad (name) recibe el valor de nombre.
La propiedad id lo que hace es darle un identificador al input para que pueda ser modificado o conectado a una propiedad de CSS o JavaScript, es recomendable darle un valor que este relacionado al tipo de dato que se está pidiendo. En el caso de los inputs también se utiliza para que el atributo for de los label apunte hacia ese mismo nombre y así puedan asociarse.

<input type='text' id='nombre' name='Nombre' />

Propiedad Disable

Este atributo sirve para deshabilitar un campo y así impide que se ingresen datos. Este atributo puede tomar dos valores true or false.

<input type='text' placeholder='No se puede modificar este campo' disable='true' />

Propiedad Readonly

Esta propiedad es parecida a disable, la diferencia es que es un campo solo para mostrar información o una entrada que no es editable, es decir, que solo se puede apreciar su lectura.

<input type='text' readonly />

Propiedad Max Length

Le especifica al navegador cuál es el máximo números de caracteres que puede dejar ingresar en el cuadro de entrada.

<input type='text' maxlength='30' placeholder='Solo 30 caracteres' />

Propiedad Value

Es asignado para darle un valor por defecto al elemento input. Es utilizado en caso de que la persona olvide llenar este cuadro de entrada y necesitemos dejar un valor específico que debemos guardar. En otras palabras, retorna el valor que contiene el input. Debemos tener mucho cuidado con autocompletar este campo ya que es el dato que se guardará en la base de datos.

<label for='nombrePrincipal'>Ingrese su nombre: </label>
<input type='text' value='Juan (Autocompletado)' id='nombrePrincipal' />

Propiedad Placeholder

Se usa para dar una pista al usuario sobre que debe completar en el campo input solicitado. El mensaje aparecerá dentro del cuadro de texto, mientras esté vacio.

<label for='direccion'>Ingrese su direccion: </label>
<input type='text' placeholder='Roca 1234' id='direccion' />

Validar los campos

El atributo required se aplica dentro del elemento input y es muy útil para realizar las validaciones de algunos campos de nuestro formulario. En el siguiente formulario de ejemplo tenemos algunos campos para completar y cada uno de ellos con el atributo required.

<form>
<fielset>
<legend>Validación de Formularios</legend>

<label for='email-usuario'>Ingrese su email: </label>
<input type='email' id='email-usuario' placeholder='Ej: Juan@gmail.com' />

<label for='edad-usuario'>Ingrese su edad: </label>
<input type='number' id='edad-usuario' placeholder='Ej: 29' />
</fielset>
</form>

Captura de datos especiales

Checkbox

El checkbox es una caja le permite al usuario activar una opción o no. Para crear un checkbox empleamos la etiqueta "input" con el atributo type=”Checkbox”. Es importante colocar una etiqueta (label) y enlazarla con el atributo for al checkbox a través de su id, donde el valor for y el valor del id deben coincidir.

Radio

El radio o radio button es una entrada que permite seleccionar exclusivamente entre un conjunto de alternativas. Para crear un radio button usamos la etiqueta "input" con el atributo type=”radio”. Es importante colocar una etiqueta (label) y enlazarla con el atributo for al checkbox con el id, donde el valor for y el valor del id deben coincidir. También, es importante colocar la propiedad de value ya que va a contener el valor de la casilla que marque el usuario.

Lista con grupos

Para crear una lista con grupos utilizamos la etiqueta select y dentro de ella, la etiqueta otpgroup para definir el grupo, por cada etiqueta otpgroup es un grupo diferente. Lista de selección múltiple

Para crear una lista de selección múltiple usamos la etiqueta select que permite seleccionar múltiples elementos. El primer elemento se selecciona con un click, pero los adicionales se seleccionan usando (ctrl+click) ó (cmd+click) según el navegador y sistema operativo.
