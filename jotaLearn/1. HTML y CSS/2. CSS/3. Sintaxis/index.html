Sintaxis de CSS

Introducción

La sintaxis de CSS se compone de tres partes principales: selectores, propiedades y valores.

  1. Selectores: Determinan a qué elementos HTML se aplicarán los estilos. Por ejemplo, un selector puede ser una etiqueta HTML específica como <p> o una clase o ID personalizados que hayas definido en tu HTML.
  2. Propiedades: Definen los aspectos específicos que se quieren estilizar, como el color del texto o el espacio alrededor de un elemento.
  3. Valores: Especifican cómo se quiere que se vea esa propiedad. Por ejemplo, el color del texto puede ser "rojo" o "#ff0000".

Ejemplo de Sintaxis

Selector de etiqueta

p {
color: red;
text-align: center;
font-size: 20px;
}

Selector de clase

.titulo {
color: blue;
text-align: left;
font-size: 30px;
}

Selector de Id

#subtitulo {
color: green;
text-align: right;
font-size: 25px;
}

En este ejemplo podemos ver e identificar los distintos selectores , valores y propiedades. Entre los selectores podemos ver el selector de etiqueta , con la etiqueta <p>, el selector de clase, “.titulo” y el selector de id, “#subtitulo”.


Por otro lado, en cuanto a propiedades podemos identificar 3: color, text-align y font-size.


Finalmente, tenemos los valores. Por un lado, valores de color como green, blue y red, valores para la alineación del texto, como center, left y right, y finalmente valores de tamaño de letra, como 20px, 30px y 25px.


Además podemos ver que los selectores se colocan antes de la llave de apertura que envuelve los estilos que se van a declarar , siendo cada combinación de propiedad y valor una declaración. Finalmente, la sintaxis concluye con la llave de cierre que envuelve las declaraciones de estilos.

Concepto Cascada

Este concepto se refiere a la posibilidad de especificar cuál estilo tiene prioridad jerárquica. Por ejemplo, si tenemos “color red” en una etiqueta “style” y “color blue” en un “archivo externo” y ambos están aplicados a la etiqueta <p> ¿Cuál debería mostrar el navegador?


Para renderizarse en el navegador, nuestra Web tiene varias fuentes de estilos que se van colocando una después de la otra. De ahí viene el término Cascade Style Sheet, debemos tener en cuenta que si tenemos un proyecto web que aún no tiene estilos definidos, prevalecerá la hoja de estilos que viene por defecto en el navegador.


Por ejemplo, cuando creamos un enlace si ya lo hemos visitado, este se pone de un color específico según los estilos predefinidos del navegador. Actualmente algunos navegadores permiten que carguemos en su configuración hojas de estilo personalizadas. Este archivo tendrá prioridad sobre los propios del navegador usado y se conoce como: hojadeestilosdeusuario.css

¿Qué pasaría si tuviéramos varios archivos CSS de autor en nuestro sitio?

La prioridad la tendría el último archivo CSS vinculado. Por ejemplo, si tuviéramos el siguiente código:

<head>
<title>Aplicando CSS</title>
<link rel='stylesheet' href='./css/styles.css' />
<link rel='stylesheet' href='./css/styles2.css' />
<link rel='stylesheet' href='./css/styles3.css' />
</head>

Los estilos que tendrían prioridad serían los del último archivo, que en este caso se llaman: './css/estilos3.css'.

Palabra reservada: !important

Hay ocasiones en las que debemos aplicar un estilo donde el navegador posiblemente no esté tomando correctamente el selector. La manera de aplicar este estilo de una forma imperativa es usando la palabra reservada ‘!important’.

body {
font-family: verdana !important;
}

Esta regla obligará a que todo el cuerpo de la página esté vinculado con el archivo CSS que la contenga y maneje la fuente verdana sobre otro cualquier tipo de fuente definido.

Prioridad de jerarquía en los estilos

Estilo en línea: atributo 'style'.

Los estilos en línea que se encuentran en la hoja de HTML son los que tienen prioridad sobre todas las demás declaraciones, debemos destacar que este tipo de estilos no son una práctica apropiada para aplicar.

Estilos en encabezado: Etiqueta <style>.

Los estilos que se encuentran en el encabezado del documento HTML tienen prioridad sobre la hoja de estilos externa pero no sobre los estilos en línea. También, debemos destacar que no es una buena práctica aplicar este tipo de estilos.

Concepto Herencia

La herencia debe entenderse con los siguientes términos: padres, hijos y hermanos. Teniendo en cuenta esta jerarquía podemos afirmar que el padre de todo el contenido es la etiqueta HTML, el cual agrupa a dos hijos que son las etiquetas de head y body. Estos últimos son hermanos porque tienen el mismo nivel jerárquico y también son padres de todos los elementos que ellos componen.


Esto nos permite dar cuenta que cuando un elemento padre tiene estilos definidos, por ejemplo, el color rojo en el texto estos estilos pasarán a todos sus hijos. Es decir, a todos los elementos que se encuentren en su interior lo que significa que estas etiquetas hijas van a heredar todos los estilos que se apliquen al elemento padre.

Ejemplo de herencia

body {
color: blue;
}

En este ejemplo estaremos aplicando color azul a todo el texto del documento, ya que, la etiqueta body es padre de todos los elementos que se van a visualizar en el navegador. Si prestamos atención, todos los elementos que vayamos a crear dentro de la estructura de estas etiquetas tomarán el comportamiento que le hemos aplicado a su etiqueta padre.

Cambiando de estilos a etiquetas hijas

Si tenemos en una etiqueta padre un estilo aplicado que nos afecta a las demás etiquetas que están dentro de su estructura, debemos recordar que CSS tiene el concepto en cascada, es decir, si aplicamos un estilo nuevo sobre el elemento hijo este tendrar la prioridad. Tomando el ejemplo anterior donde tenemos estilos sobre el elemento body, vamos a modificar un elemento hijo (ej: <h1>) a un color negro.

body {
color: blue;
}

h1 {
color: black;
}

En este caso específico, todo el documento tendrá el texto azul excepto el elemento <h1> que será de color negro, porque, respeta los estilos en cascada. Es decir, cómo se ubica declarado más cerca de la parte inferior de la última línea será el estilo que pisa al que está declarado anteriormente, por más que el elemento declarado sea el elemento padre.
