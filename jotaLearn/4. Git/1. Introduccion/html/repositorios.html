Introducción

Un repositorio es el lugar dentro del sistema de control de versiones en donde guardamos las diferentes versiones de nuestro archivo. Un ejemplo coloquial para esta definición sería que creamos un álbum de fotos y luego cada semana cargamos una foto. Esto se traduce a que nuestro álbum es la carpeta del repositorio y las fotos son las versiones del documento.


Git, almacena los archivos dentro de su nube para cuando quieras acceder a una versión en específica, esta estará disponible. Cabe destacar que los repositorios pueden tener múltiples tipos de archivos dentro de sus carpetas, hasta, otras carpetas con otra estructura de archivos.

Repositorio Remoto: Github

Sabiendo el concepto de repositorio ahora, volvamos a repasar los conceptos de esta herramienta llamada github. Es una plataforma en línea que brinda servicios de alojamiento de control de versiones y colaboración para proyectos de software. GitHub es esencialmente un servicio basado en la web para alojar y compartir repositorios de Git, lo que permite a los desarrolladores trabajar juntos en proyectos de software de manera más eficiente.


Además de alojar repositorios Git, GitHub ofrece una amplia variedad de herramientas y características para colaborar en proyectos, incluyendo seguimiento de problemas, pull requests, wikis, estadísticas y más. GitHub es una plataforma ampliamente utilizada en la comunidad de desarrollo de software y es uno de los servicios de control de versiones más populares en la actualidad.

Creando un repositorio remoto

Para crear un nuevo repositorio en github, debemos seguir los siguientes pasos:

  1. Regístrate o Inicia sesión en GitHub: Inicia sesión en tu cuenta de GitHub o crea una nueva cuenta si aún no tienes una - Crea tu cuenta aquí.
  2. Iniciar la creación: Una vez registrada la cuenta o iniciada la sesión, haz clic en el botón "New repository" en la página principal de tu perfil de GitHub.

  3. Configura el repositorio: Da un nombre y descripción al repositorio y selecciona la visibilidad del repositorio (público o privado). También puedes elegir inicializar el repositorio con un archivo README ya creado.
  4. Crear repositorio: Haz clic en el botón "Create repository" para crear el repositorio.

Repositorio Local: Configurando Git.

Una vez realizados los pasos anteriormente mencionados, ya tenemos nuestro repositorio listo para alojar un proyecto. Veamos cuales son los pasos a realizar para empezar a subir archivos desde nuestra área de trabajo local a dicho repositorio remoto:

1. Descargar e instalar Git

Descargamos e instalamos la última versión de Git para nuestro sistema operativo desde la página oficial de Git - Descarga e instala git aqui


IMPORTANTE: Durante la instalación nos aseguramos de que se designe la rama “main” como la rama principal de nuestro proyecto. Para ello, verificamos que al instalar , en el paso que corresponde al ajuste de la rama inicial de un proyecto.

2. Configurar Identidad

Una de las maneras de iniciar git es presionando el click derecho sobre el entorno al cual vamos acceder en consola y la opción a abrir es git bash. Una vez hecho este paso para corroborar si el programa está funcionando correctamente usaremos el comando git –version el cual nos retornará la versión de git en la consola si todo funciona correctamente.


Nuestro siguiente paso es acceder como usuario al programa, para que este pueda identificarnos y poder usar los directorios que tenemos almacenados, para esto debemos saber 2 nuevos comandos que son: tu nombre de usuario y dirección de correo electrónico. Es importante que el email que utilices sea el mismo con el cual te registraste en GITHUB.


Comandos

git config --global user.name 'Tu nombre'
git config --global user.email 'Tu mail que usaste en github'

Ejemplo Práctico

juand@DESKTOP-6M2PRO8 MINGW64 ~
$ git --version
git version 2.35.1windows.2

juand@DESKTOP-6M2PRO8 MINGW64 ~
$ git config --global user.name 'juan'

juand@DESKTOP-6M2PRO8 MINGW64 ~
$ git config --global user.mail 'juan@mail.com'

Configurar rama principal

En caso de que no lo hayas realizado en la instalación, puedes colocar el siguiente comando en la consola para poder configurar la rama main como la rama principal de todos los proyectos que inicialices de ahora en más en tu área de trabajo local

juand@DESKTOP-6M2PRO8 MINGW64 ~
$ git config --global init.defaultBranch main

Una vez realizados estos 3 pasos, ya podemos comenzar a ver los comandos que nos permitirán subir un proyecto que tengamos en una carpeta local al almacenamiento en la nube que nos otorga Github.

Commits

Más adelante, estaremos viendo una acción que desarrollan los repositorios llamada commit. Siguiendo con el ejemplo de los álbumes y las fotografías, cada fotografía recibe un “commit”. Esto representa una versión, es muy importante saber que cada ‘commit’ recibe un identificador único. Volvemos a remarcar que, el álbum donde se almacenan los commits es un repositorio.
